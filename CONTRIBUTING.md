# Contributing to VeloGuardian Community

Thank you for your interest in improving VeloGuardian products. This repository is the place to report bugs, request features, and discuss VeloGuardian DNS — our free DNS filtering appliance.

## What you can contribute

### Bug reports

If you've found a bug in VeloGuardian DNS, please [open a bug report](https://github.com/veloguardian/community/issues/new?template=dns-bug-report.yml). Include:

- Your DNS appliance version (shown in the dashboard footer)
- Steps to reproduce the issue
- Expected vs actual behavior
- Screenshots if applicable

### Feature requests

Have an idea for VeloGuardian DNS? [Submit a feature request](https://github.com/veloguardian/community/issues/new?template=dns-feature-request.yml). We review all requests and prioritize based on community interest.

### Discussions

For general questions, setup help, or open-ended feedback, use [GitHub Discussions](https://github.com/veloguardian/community/discussions).

### Documentation improvements

If you spot errors or have improvements for the DNS documentation on our website, open an issue describing the problem and we'll update it.

## What this repo does NOT contain

This is a community feedback repository, not a code repository. VeloGuardian products are proprietary and closed-source. Pull requests with code changes will not be accepted, but PRs improving documentation files in this repo (like the changelog or known issues) are welcome.

## Scope

This repository covers **VeloGuardian DNS** community feedback. For VPN and NetGuard support, please contact us at [contact@veloguardian.com](mailto:contact@veloguardian.com) or visit [veloguardian.com/support](https://veloguardian.com/support.html).
